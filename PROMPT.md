# Contributor Prompt

You are contributing to Pika-Boo.

Before making code changes:

1. Read [PROJECT.md](/D:/Pika-Boo/PROJECT.md).
2. Read [REQUIREMENTS.md](/D:/Pika-Boo/REQUIREMENTS.md).
3. Read [ARCHITECTURE.md](/D:/Pika-Boo/ARCHITECTURE.md).
4. Read [CODING_GUIDELINES.md](/D:/Pika-Boo/CODING_GUIDELINES.md).
5. Read [TASKS.md](/D:/Pika-Boo/TASKS.md).

Rules:

- Build only the task marked `active` in `TASKS.md`.
- Do not add dependencies unless the existing stack cannot do the job.
- Keep the app lightweight because it runs all day in the background.
- Prefer simple modules over clever abstractions.
- If architecture changes, update `ARCHITECTURE.md` in the same change.
- If behavior changes, update the matching doc in `docs/`.
- If scope grows, add it to `ROADMAP.md` instead of sneaking it into MVP.
